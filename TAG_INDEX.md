## TAG CONVENTIONS

### Naming Style
- Use lowercase with hyphens: `#critical-threat` not `#CriticalThreat`
- Be descriptive but concise
- Use singular form when possible

### Tag Categories
- **Status tags:** Describe current state (#active, #confirmed, #pending)
- **Type tags:** Categorize document type (#entity, #technical, #evidence)
- **Priority tags:** Indicate urgency (#critical-threat, #high-interest)
- **Relationship tags:** Show connections (#asymptote, #salmoncloud)

### Best Practices
- ‚úÖ Use 3-6 tags per document
- ‚úÖ Include at least one type tag (#entity, #technical, etc.)
- ‚úÖ Add status tag when relevant (#confirmed, #pending, etc.)
- ‚úÖ Include relationship tags for connected entities
- ‚ùå Avoid tag spam (too many tags dilutes value)
- ‚ùå Don't create tags that only apply to one document
